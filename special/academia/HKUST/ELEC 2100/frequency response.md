# frequency response

- HKUST ELEC 2100

<!-- check: ignore-file[two_sided_calc_warning]: concept-focused prompts intentionally use descriptive wording rather than repeating every symbol on the left-hand side -->

Frequency response is the frequency-domain description of how an LTI system processes sinusoidal content. In ELEC 2100 it is the bridge from Fourier transforms of signals to transform-domain analysis of systems: once the input spectrum is known, the system acts by frequency-dependent scaling and phase shifting rather than by repeated time-domain convolution. Pair this note with [Fourier transform](Fourier%20transform.md): that note is the signal-side home for transform pairs and property derivations, while this note is the system-side home for interpreting how those spectral components are processed by LTI systems and filters.

## time-domain and frequency-domain zero-state viewpoints

The zero-state response of an LTI system can be organized in two equivalent but differently useful ways. In the time domain, `convolution.md` shows that $y(t)=x(t)*h(t)$, so the output is assembled from delayed copies of the impulse response. In the frequency domain, the Fourier transform turns that convolution into multiplication, giving $Y(\omega)=H(\omega)X(\omega)$. The two pictures are not competing methods; they are the same zero-state law written in two coordinate systems.

This note therefore starts where the convolution note leaves off. Convolution is the better language when support, overlap geometry, delays, and transient shape are the main story. Frequency response is the better language when one wants to know which sinusoidal components are amplified, attenuated, or phase shifted. The bridge between the notes is exactly the Fourier transform of the impulse response: $H(\omega)=\mathcal{F}\{h(t)\}$.

The practical memory cue is simple. If the question is "what does one impulse or one pulse do in time?", think in terms of $h(t)$ and convolution. If the question is "what does the system do to each frequency component?", think in terms of $H(\omega)$ and multiplication. The mathematics is consistent because both viewpoints describe the same LTI system.

## complex exponentials as LTI eigenfunctions

If the zero-state impulse response is $h(t)$ and the input is $x(t)=e^{j\omega t}$, then $y(t)=(h*x)(t)=\int_{-\infty}^{\infty}h(\tau)e^{j\omega(t-\tau)}\,d\tau=e^{j\omega t}\int_{-\infty}^{\infty}h(\tau)e^{-j\omega\tau}\,d\tau$. The integral depends on $\omega$ but not on $t$, so the output has the form $y(t)=H(\omega)e^{j\omega t}$, where $H(\omega)=\int_{-\infty}^{\infty}h(t)e^{-j\omega t}\,dt$ is the Fourier transform of the impulse response.

This is why complex exponentials are eigenfunctions of LTI systems. The oscillation frequency is preserved, and only a frequency-dependent complex scalar is attached. For a general input spectrum $X(\omega)$, the zero-state output spectrum is $Y(\omega)=H(\omega)X(\omega)$, which is exactly the convolution theorem viewed as independent processing of each spectral component.

## magnitude response and phase response

Write the system function as $H(\omega)=|H(\omega)|e^{j\phi(\omega)}$. Then a sinusoidal component keeps its frequency but changes amplitude and phase. If $x(t)=A\cos(\omega t+\theta)$, the corresponding output component is $y(t)=A|H(\omega)|\cos(\omega t+\theta+\phi(\omega))$. So magnitude response tells how strongly the frequency passes, while phase response tells how that frequency is shifted in phase.

There is also an important timing interpretation. A pure delay by $t_0$ has frequency response $e^{-j\omega t_0}$, so its phase is the straight line $-\omega t_0$. Therefore a phase curve that is approximately linear in $\omega$ behaves like an approximate delay over the frequency range where that straight-line approximation is good. This is why phase response is not just an extra decoration on top of magnitude; it tells whether different spectral components leave the system with different effective timing shifts.

This is the system-analysis meaning of filter language. A low-pass filter has large $|H(\omega)|$ near $\omega=0$ and small $|H(\omega)|$ for large $|\omega|$. A high-pass filter does the reverse. A band-pass filter emphasizes one middle frequency range. The phase response matters because two systems with the same magnitude response can still reshape a waveform differently if their phases differ, and a nearly linear phase law corresponds to an approximately constant delay.

## ways to determine the system function

There are three practical routes to $H(\omega)$. First, if the impulse response is known, compute $H(\omega)=\mathcal{F}\{h(t)\}$. Second, if the input-output model is $a_n y^{(n)}+\cdots+a_0 y=b_m x^{(m)}+\cdots+b_0 x$, replace each derivative by multiplication by $j\omega$ to obtain $H(\omega)=\frac{b_m(j\omega)^m+b_{m-1}(j\omega)^{m-1}+\cdots+b_0}{a_n(j\omega)^n+a_{n-1}(j\omega)^{n-1}+\cdots+a_0}$. Third, in circuit problems, replace reactive elements by their frequency-domain impedances and solve for the output-to-input ratio directly.

These three routes are the same idea seen through different representations. The impulse-response route emphasizes structure, the differential-equation route emphasizes operator algebra, and the circuit route emphasizes engineering computation.

## first-order RC low-pass filter

The standard first-order RC low-pass filter is wired as follows. Start from the input source positive terminal and go through the resistor $R$ to a single intermediate node. That intermediate node is the output node. From that same node, connect the capacitor $C$ down to ground. The source negative terminal is also tied to ground, so the resistor and capacitor form one series path from source to ground, with the output measured between the intermediate node and ground. If the input voltage is $x(t)$, the resistor voltage is $v_R(t)$, the capacitor voltage is $v_C(t)$, and the output is defined as $y(t)=v_C(t)$, then Kirchhoff's voltage law around the loop gives $x(t)=v_R(t)+v_C(t)=v_R(t)+y(t)$. This is the starting equation: the source voltage is split between the resistor drop and the capacitor drop.

To turn KVL into a system equation, use the constitutive laws of the elements. The resistor law is $v_R(t)=Ri(t)$, and the capacitor law is $i(t)=C\,dy(t)/dt$ because the capacitor current is proportional to the rate of change of its voltage. Substituting these into KVL gives $x(t)=RC\,dy(t)/dt+y(t)$, or equivalently $RC\,dy(t)/dt+y(t)=x(t)$. So the RC low-pass filter is a first-order linear differential-equation system whose output cannot jump arbitrarily fast unless the capacitor voltage changes arbitrarily fast.

This differential equation already explains the low-pass behavior physically. At low frequency, the capacitor voltage has time to follow the input, so the output remains close to the input. At high frequency, rapid oscillation requires large capacitor current, which produces a larger resistor drop and leaves less of the input across the capacitor. In plain circuit language: the capacitor smooths fast variation and preserves slow variation.

In the frequency domain, one can derive the same result from impedances. The capacitor impedance is $1/(j\omega C)$, so voltage division gives $H(\omega)=\frac{1/(j\omega C)}{R+1/(j\omega C)}=\frac{1}{1+j\omega RC}$. If $\alpha=1/(RC)$, then $H(\omega)=\frac{\alpha}{\alpha+j\omega}$, the magnitude is $|H(\omega)|=\frac{1}{\sqrt{1+(\omega RC)^2}}=\frac{\alpha}{\sqrt{\alpha^2+\omega^2}}$, and the phase is $\phi(\omega)=-\operatorname{atan2}(\omega,\alpha)$.

The phase formula deserves its own interpretation. Since $\phi(\omega)=-\operatorname{atan2}(\omega,\alpha)$, the phase starts at $0$ when $\omega=0$ and becomes more negative as frequency increases. Near low frequency, where $|\omega|\ll \alpha$, one may use $\operatorname{atan2}(\omega,\alpha)\approx \omega/\alpha=\omega RC$, so $\phi(\omega)\approx-\omega RC$. That is approximately a straight line with slope $-RC$, so low-frequency components of the RC filter behave approximately like a signal delayed by about $RC$ seconds. This is only an approximation over the low-frequency region, not an exact all-frequency delay, because the true phase curve bends away from a straight line as $|\omega|$ grows.

One can say the same thing in slightly more precise language through group delay. Since $\phi(\omega)=-\operatorname{atan2}(\omega,\alpha)$, the local delay-like quantity is $-d\phi/d\omega=\alpha/(\alpha^2+\omega^2)=RC/(1+(\omega RC)^2)$. So the effective delay is close to $RC$ near the origin and becomes smaller at higher frequency. That is why the RC filter slightly delays slowly varying waveform features, but does not preserve all frequency components with one common exact delay.

At $\omega=0$, the gain is $1$, so DC passes unchanged. As $|\omega|\to\infty$, the gain tends to $0$, so rapid oscillations are attenuated. The cutoff frequency is the $3\text{ dB}$ or half-power point. The phrase half-power matters: if the same load interpretation is used, output power is proportional to amplitude squared, so $|H(\omega_c)|^2=1/2$ means the output power has dropped to one half of the passband value. Therefore the amplitude ratio at cutoff is $|H(\omega_c)|=1/\sqrt{2}$, which gives $\omega_c=1/(RC)=\alpha$. So the time constant $RC$ and the low-pass transition scale are reciprocal ideas: larger $RC$ means slower time-domain charging and a narrower passband.

This real RC filter should also be compared with the ideal low-pass filter directly in formulas. The RC filter has $H_{\mathrm{RC}}(\omega)=\frac{1}{1+j\omega RC}$ and $|H_{\mathrm{RC}}(\omega)|=\frac{1}{\sqrt{1+(\omega RC)^2}}$, so its magnitude decays smoothly and never becomes exactly zero at any finite frequency. The ideal low-pass filter instead has $H_{\mathrm{LP}}(\omega)=1$ for $|\omega|<\omega_c$ and $0$ for $|\omega|>\omega_c$, so its magnitude is a brick-wall function $|H_{\mathrm{LP}}(\omega)|=1$ inside the passband and $0$ outside. Their phases also differ sharply: the RC filter has the continuous phase law $\phi_{\mathrm{RC}}(\omega)=-\operatorname{atan2}(\omega,1/RC)$, whereas the ideal low-pass filter has zero phase in the simplest noncausal symmetric form. So the RC filter is causal, physically realizable, and first-order, but its transition from passband to stopband is gradual; the ideal low-pass filter has perfectly sharp frequency selection, but that sharpness forces a two-sided sinc impulse response and therefore noncausality. The RC circuit is the practical engineering low-pass prototype, while the ideal low-pass filter is the conceptual frequency-selective limit.

## ideal low-pass, high-pass, and band-pass filters

The ideal low-pass filter can be written compactly as $H_{\mathrm{LP}}(\omega)=\operatorname{rect}(\omega/(2\omega_c))$, meaning $H_{\mathrm{LP}}(\omega)=1$ for $|\omega|<\omega_c$ and $0$ outside. To derive its impulse response from a standard pair, start with $\operatorname{rect}(t/\tau)\longleftrightarrow \tau\operatorname{sinc}_{\pi}(\omega\tau/(2\pi))$, where $\operatorname{sinc}_{\pi}(x)=\sin(\pi x)/(\pi x)$. Then apply the explicit duality rule $f(t)\longleftrightarrow F(\omega)\Rightarrow F(t)\longleftrightarrow 2\pi f(-\omega)$. Since $\operatorname{rect}$ is even, duality gives $\tau\operatorname{sinc}_{\pi}(t\tau/(2\pi))\longleftrightarrow 2\pi\operatorname{rect}(\omega/\tau)$. Now choose $\tau=2\omega_c$ and divide by $2\pi$. The result is $\operatorname{rect}(\omega/(2\omega_c))\longleftrightarrow (\omega_c/\pi)\operatorname{sinc}_{\pi}(\omega_c t/\pi)=\sin(\omega_c t)/(\pi t)$. Therefore $h_{\mathrm{LP}}(t)=\frac{\omega_c}{\pi}\operatorname{sinc}_{\pi}(\omega_c t/\pi)=\frac{\sin(\omega_c t)}{\pi t}$. Because this kernel extends over all time, the ideal low-pass filter is noncausal and cannot be realized exactly in real time. The main intuition is that perfectly sharp truncation in frequency forces infinite spread in time.

The ideal high-pass filter is best understood as the complement of that low-pass selector. Since $H_{\mathrm{HP}}(\omega)=1-H_{\mathrm{LP}}(\omega)$, the time-domain kernel is immediately $h_{\mathrm{HP}}(t)=\delta(t)-\frac{\sin(\omega_c t)}{\pi t}$. So the quickest memory rule is not to memorize a fresh transform pair from nowhere, but to remember that ideal high-pass means "identity minus ideal low-pass".

For a symmetric ideal band-pass filter, it is often better to describe the passband by center frequency and half-bandwidth. If the lower and upper cutoffs are $\omega_{\mathrm{L}}=\omega_0-\omega_c$ and $\omega_{\mathrm{U}}=\omega_0+\omega_c$, then $H_{\mathrm{BP}}(\omega)=\operatorname{rect}((\omega-\omega_0)/(2\omega_c))+\operatorname{rect}((\omega+\omega_0)/(2\omega_c))$, which is equivalent to saying the filter passes $\omega_0-\omega_c<|\omega|<\omega_0+\omega_c$. Starting from the ideal low-pass kernel and applying the frequency-shift rule, one gets $h_{\mathrm{BP}}(t)=2\cos(\omega_0 t)\,\frac{\omega_c}{\pi}\operatorname{sinc}_{\pi}(\omega_c t/\pi)=\frac{\sin((\omega_0+\omega_c)t)-\sin((\omega_0-\omega_c)t)}{\pi t}$. This form is especially useful in channel-selection problems: the cosine places the passband around the center frequency, while the sinc envelope sets the channel width.

This also gives a strong interpretation for ideal band-pass behavior. An ideal low-pass filter keeps frequencies near the origin. An ideal band-pass filter is the same rectangular selection window shifted away from the origin to surround $\pm\omega_0$. In time, that shift becomes multiplication by $\cos(\omega_0 t)$, so the impulse response looks like a carrier-frequency oscillation under a sinc-shaped envelope. That is why ideal demultiplexing formulas naturally involve band-pass filters before remodulation.

Compared with the first-order RC low-pass filter, the ideal low-pass filter is a standard of comparison rather than a practical one-pole circuit. In formulas, $|H_{\mathrm{RC}}(\omega)|=\frac{1}{\sqrt{1+(\omega RC)^2}}$ decays continuously from $1$ toward $0$, while $|H_{\mathrm{LP}}(\omega)|$ is exactly $1$ in the passband and exactly $0$ in the stopband. The price of the ideal sharp edge is noncausality; the price of the RC filter's realizability is imperfect selectivity and a gradual transition band.

This comparison is the right way to organize "ideal versus practical" thinking. Ideal filters are frequency-domain limit objects: perfect selection, abrupt edges, and impulse responses that extend forever in time. Practical first-order circuits are realizable causal approximations: finite-order, physically buildable, and easy to analyze, but only gradually selective. So the engineering question is not which one is "better" in the abstract, but which tradeoff between selectivity, realizability, and phase behaviour matters in the application.

A standard lecture-style example starts from the plotted magnitude and phase of an ideal high-pass filter rather than from a memorized formula. If the magnitude is $0$ for $|\omega|<\omega_c$ and $1$ for $|\omega|>\omega_c$, while the passband phase is taken as $0$, then the frequency response is $H_{\mathrm{HP}}(\omega)=1-\operatorname{rect}(\omega/(2\omega_c))$. The inverse transform is then immediate from complement thinking: $1\longleftrightarrow\delta(t)$ and $\operatorname{rect}(\omega/(2\omega_c))\longleftrightarrow \sin(\omega_c t)/(\pi t)$, so $h_{\mathrm{HP}}(t)=\delta(t)-\sin(\omega_c t)/(\pi t)$.

The parallel examples are worth keeping side by side. For the ideal low-pass filter itself, $H_{\mathrm{LP}}(\omega)=\operatorname{rect}(\omega/(2\omega_c))$ gives $h_{\mathrm{LP}}(t)=\sin(\omega_c t)/(\pi t)$. For the ideal band-pass filter centered at $\omega_0$ with half-bandwidth $\omega_c$, $H_{\mathrm{BP}}(\omega)=\operatorname{rect}((\omega-\omega_0)/(2\omega_c))+\operatorname{rect}((\omega+\omega_0)/(2\omega_c))$ gives $h_{\mathrm{BP}}(t)=2\cos(\omega_0 t)\,\sin(\omega_c t)/(\pi t)=\bigl[\sin((\omega_0+\omega_c)t)-\sin((\omega_0-\omega_c)t)\bigr]/(\pi t)$. So the quickest memory rule is: ideal low-pass gives sinc, ideal high-pass gives impulse minus sinc, and ideal band-pass gives a modulated sinc.

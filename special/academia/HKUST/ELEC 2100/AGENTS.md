# ELEC 2100 agent instructions

- Keep this course scaffold logistics-heavy until lecture PDFs, tutorial sheets, or worked solutions are actually ingested; do not invent transform derivations or topic notes from vague source-unit titles alone.
- Keep `index.md` pages lean: the course root should give the overview, folder indexes should summarize navigation, and leaf indexes should usually hold only minimal logistics.
- The chosen sections are L1 for lecture, T2 for tutorial, and LA3 for lab; default schedule metadata should follow those sections unless a page is intentionally summarizing all official sections in parallel.
- When tutorial routing is kept in the course root instead of separate child pages, represent it as chronological `## week N tutorial` sessions using the chosen T2 stream; do not collapse tutorial coverage into one bulk tutorial-summary section.
- When the course root keeps week-by-week chronology, include `## week N lab` sections for every teaching week using the chosen LA3 timeslot, even when the content is only `status: no class` or a public-holiday marker.
- For Canvas-derived assignment-style leaf indexes in this course (including `labs/lab X/index.md` and similar deliverable pages, but not quizzes), keep the visible Canvas page wording verbatim. Preserve color with `<span style>`, use Markdown for bold, italics, links, paragraphing, and line breaks, and structure the page as: index metadata (with no extra `---` after the parent line), `## children`, `## description`, `## attachments`, `## submission`, and `## solution`. Inside `## description`, do not keep the Canvas title as a heading; add it as the first list item in the exact form `- title: <Canvas title header verbatim>`, then continue with the other Canvas properties. Omit generic `## logistics` and `## overview` sections, include an explicit attachments list pointing to the local `attachments/` files, and leave `submission` / `solution` empty until the user provides content.
- For homework pages in this course when only the problem handout should stay public, keep the statement files in the public `attachments/` folder and move only submission and solution artifacts into the mirrored `private/.../assignments/` subtree. If the task still wants the public page to behave as if those artifacts were colocated, keep ordinary relative links such as `[submission.pdf](submission.pdf)` or `[HW1_Sol.pdf](solution/HW1_Sol.pdf)` in `## submission` / `## solution` rather than rewriting them to `private/`. Format `## solution` exactly like `## attachments` with plain markdown file bullets, and reserve the more advanced `- file:` plus nested metadata bullets for `## submission` only. If the Canvas wording mentions a file that is genuinely missing from the archived export, preserve that wording in `## description` without inventing a fake file.
- For Canvas-derived metadata in this course, normalize any date, datetime, or duration value to ISO 8601 with timezone where applicable. Use one ISO datetime for `Due` or `locked at`, use an ISO datetime range plus `, <ISO duration>` when both availability endpoints are known, and use a single ISO datetime with an explicit key such as `Available until` when only the closing endpoint is archived. Canvas starts use seconds `:00`; Canvas ends use seconds `:59`.
- In this course, that normalization applies only to metadata fields. Keep the ordinary Canvas description prose verbatim, including human-readable date text in sentences such as `This assignment was...` and the colored `Due on` notice line.
- For Canvas quiz conversions in this course, keep the public quiz page in `questions/quiz N.md` and the private archival mirror in `private/.../questions/quiz N.md`. Public quiz pages should use `## hints` and recast the quiz into study-oriented recognition rules, conceptual prompts, and compact reminders. Keep those hints in the same order as the archived/private questions even after later rewrites. Keep them close enough to the original question that the reader can still map them back quickly: preserve the option family, theorem name, transform form, timing pattern, or essential band-edge geometry when that anchor helps, but do not copy the full official prompt verbatim. When the user asks for more context, put that extra context mainly on the left-hand side of the hint card by using option-family cues, mutated-but-equivalent givens, or the exact band/timing landmarks that drive the answer. Use alternate-number toy equations only when they genuinely clarify the same method better than a direct but still non-verbatim hint would. Private quiz pages should preserve the official prompt text in `## content`, usually as blockquotes, and should normally start directly with the questions rather than with a stock explanatory paragraph. When the archive shows only a checked option without explicit grading correctness, label it as an archived selection rather than a confirmed solution. If the user explicitly confirms that the checked options are correct, keep the prompt, `Solution:`, and `Explanation:` inside the same quoted question block rather than moving the answer outside the quote.
- For image-based private quiz questions in this course, extract the actual figures from the archived Canvas HTML into `private/.../questions/attachments/` when the payload is present, preserve the original Canvas-exported filenames such as `Q1_1.jpg` for traceability, and embed those images directly in the archival Markdown. Do not leave raw placeholder text like `prompt figure: Q1_1.jpg` when the archived HTML contains an extractable image payload. Even after embedding the real figure, keep a cloze-rich textual explanation of the method, landmark, or contrast that makes the answer correct, and if the `Solution:` line itself includes the figure then keep at least one cloze on that `Solution:` line too. Keep the public hint pages conceptual unless the user explicitly asks to expose the official figures there too.

State the stage-by-stage signal-processing logic: the carrier multiplication or other operation, where the shifted bands land, and which replica the LTI system preserves or rejects.

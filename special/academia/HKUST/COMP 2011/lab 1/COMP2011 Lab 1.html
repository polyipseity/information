Tasks

Compute the perimeter of a rectangle given the width and the height

Ask the user to input two integers. These variables represent the width and height of a rectangle. Calculate the perimeter of the rectangle and print the result to the screen. Note that you have to check whether width and height are positive before computing the result. If the width or height is zero or negative, print an error message to tell the user that the input has an error and stop the program.

In this lab, you can assume the input values are integers. In other words, the user won't enter numbers with decimal places. For example, 3.14 is an invalid input value.

The template code and the test cases can be downloaded: lab1_template.zip.

  • Download and unzip the above zip file.
    • In Windows, you should not double-click the zip file. You need to right-click and extract the file
    • In Mac, double-clicking the zip file should automatically extract the files to a new folder
  • Open VSCode and choose File > Open Folder...
    • For details, please refer to VSCode Guildeline: Compiling and running your first program

You should complete the TODO part of the main function in lab1.cpp: 

int main()
{
    int width, height; // Define 2 variables that will hold the width and height respectively

    cout << "Please input the width of the rectangle: ";
    cin >> width;
    cout << "Please input the height of the rectangle: ";
    cin >> height;

    // TODO: complete your program here

    return 0;
}

Checking Before Submission

Please compare your outputs with the desired outputs:

  • In VSCode, please open the terminal by one of the following two ways:
    • Press the key Ctrl+Shift+` in Windows/Linux, or Control+Shift+Backquote in Mac
    • From the menubar, select Terminal > New Terminal
    And then run this code to compile your code: 
    g++ -std=c++11 lab1.cpp -o lab1
  • You need to redirect the input of executable program to the txt test case files we provided, and redirect the output to different txt files. Here are the commands:
    • In MacOS or Linux: 
      ./lab1 < testcase/input1.txt > myOutput1.txt
    • In Windows: 
      Get-Content testcase/input1.txt | ./lab1 > myOutput1.txt

    Please replace the filenames accordingly if you would like to check other output files.

  • After that, you can compare your own output text files with the desired outputs one by one. You can compare any two files by VSCode:
    • Right click the first file you want to compare in VSCode Explorer, choose Select for Compare
    • Right click the second file you want to compare in VSCode Explorer, choose Compare with Selected

Grading Scheme

Each lab carries 3 points in total.

  • A student gets 1 point for attendance.
  • A student gets 1 point for a successful ZINC submission (lab1.cpp, the filename is case-sensitive)
    • The ZINC submission deadline is 10 minutes after your lab class. For example, if the lab end time is 12:50PM, the ZINC submission deadline is 01:00PM (i.e., 10 minutes after 12:50PM) on the same day.

    • The list of selected students will only be shown during the lab class.
    • If the student is not selected for the Q&A, then the ZINC submission will count for 2 points.
  • The filename is lab1.cpp, and submit it to ZINC. 50% penalty will be given if your filename is wrong.

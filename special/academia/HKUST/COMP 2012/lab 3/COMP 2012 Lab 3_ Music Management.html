Lab 3 Music Management

Overview

In previous Lab2, we have practiced with constructors and destructors. In this lab, we will continue to work with them. We are going to build a music management system, which contains a class Singer to represent a singer, a class Song to represent a song, a class Playlist to represent a playlist of a user, and a class MusicManager to manage them.

To make the things easier, we provide some basic implementation of the classes Singer and Song in the skeleton files. 

class Singer {
  private:
      string name;
      string nationality;
  public:
      Singer(const string& name , const string& nationality);
      Singer(const Singer&) = delete;
      string getName() const;
      string getNationality() const;
};

class Song {
  private:
      string name;
      Singer* singer;
  public:
      Song(const string& name, Singer* singer);
      Song(const Song&) = delete;
      string getName() const;
      const Singer* getSinger() const;
};

The class Singer represents a singer, which has a name and his/her nationality. The class Song represents a song, which has a name and a Singer who sings the song.

Your task is to implement some member functions of the class Playlist and MusicManager. The definition of the class Playlist is as follows: 

class PlayList {
  private:
      string username;
      const Song** songs; // dynamic array of pointers to constant Song object
      const int capacity;
  public:
      PlayList(const string& username, int size=100);
      PlayList(const string& username, const PlayList& playlist, int size=100);
      ~PlayList();
      void addSong(const Song* song);
      void removeSong(const string& name);
      string getUsername() const;
      void display() const;
  };

The definition of the class MusicManager is as follows: 

class MusicManager {
  private:
      PlayList** user_list; // dynamic array of pointers to PlayList object
      Singer** singer_list; // dynamic array of pointers to Singer object
      Song** song_list; // dynamic array of pointers to Song object
      const int user_capacity, singers_capacity, songs_capacity;
      int num_of_user, num_of_singers, num_of_songs;
  
  public:
      MusicManager(int user_max, int singers_max, int songs_max);
      ~MusicManager();
      void createSinger(const string& name, const string& nationality);
      void createSong(const string& name, const string& singer_name);
      void createPlaylist(const string& user_name);
      void createPlaylistFromOther(const string& des_user_name, const string& src_user_name);
      void displayPlaylist(const string& user_name) const;
      void addSongToPlaylist(const string& user_name, const string& song_name);
      void removeSongFromPlaylist(const string& user_name, const string& song_name);
  };

The detailed requirements are described in the Lab Tasks section.

Lab Tasks

Your task is to implement some member functions in playlist.cpp and manager.cpp (TODO 1 ~ TODO 7).

TODO 1: MusicManager constructor.
You need to initialize the data members user_list, singer_list and song_list. The parameters represent the capacity of the 3 lists, which must be used to construct the lists. Capacity of the lists is fixed and cannot be changed after the construction. Number of users, singers and songs should be initialized to 0.
TODO 2: Add a song into the MusicManager.
You need to create a new song with the specified name and singer, and add it to the song_list of the MusicManager. In this lab, you can assume that the number of elements in a list will not exceed the capacity.
TODO 3: PlayList constructor.
You need to implement the PlayList(const string&, const PlayList&, int) constructor. The constructor will copy from a given PlayList object. If the capacity of this new PlayList object is larger than the capacity of the given PlayList object, first copy the dynamic array of addresses and then initialize the remaining pointers by nullptr. Otherwise, just copy the dyanmic array of addresses.
TODO 4: Add and Remove Songs of a Playlist.
You need to implement the addSong and removeSong functions of the PlayList class.
addSong will store the address of a Song object to the first available array element (i.e. nullptr) of the dynamic array. Do nothing if there is no available element.
removeSong will remove the found song by shifting the pointers after the found song up. Do nothing if the song is not found.
TODO 5: Copy the playlist from your friend.
In skeleton code, we provide the createPlaylist function, which creates a new playlist for the user user_name. However, in some cases, we want to create a new playlist for a user by copying the playlist of another user. Therefore, you need to implement the createPlaylistFromOther function to achieve this. You need to create a new playlist for the user des_user_name by copying the playlist of the user src_user_name.
TODO 6: Build the destructors.
You need to implement the destructors of the PlayList and MusicManager classes.
TODO 7: Compile your code with Makefile
In last lab, we have introduced how to use Makefile to compile your code. In this lab, there are multiple source files, including main.cpp, playlist.cpp, manager.cpp, singer.cpp, song.cpp and library.cpp. It is not convenient to compile them one by one.
Therefore, you can write a Makefile to compile your code in this lab. You can refer to the examples in Tutorial 2, or you can modify the list of cpp files assigned to the SRCS variable in the "Makefile" from Tutorial 2 Program.

Compile and Test

After you have finished all tasks (TODO 1 ~ TODO 7), you can test your program by typing make in the terminal to compile the files. Then you can run the program by typing ./lab3.exe in the terminal. Your program must be able to re-generate exactly the same output as the sample output that has been shown in the overview section. You should read the main.cpp file carefully to understand how the sample output is generated.

Deadline and Guidelines

The lab assignment is due 10 minutes after the end of your lab session.
We will use an online grading system, ZINC, to grade your lab work. Please upload the following files as a .zip file to ZINC: 
- playlist.cpp
- manager.cpp
Note that you do not need to submit the makefile.
Please remember to zip the files directly, NOT zip a folder with the required files. After submitting, please review the ZINC report to make sure your submission is parsed correctly. You can make multiple submissions to ZINC before the deadline. Only the last submission will be graded.

Note: The ZINC score report contains 3 test cases numbered #1, #2 and #3. There are 3 additional test cases #11, #12 and #13, which are the same as the first 3 test cases, but the program is compiled using the -fsanitize flag. You will lose points for these test cases if your program contains memory leak or other runtime issues.

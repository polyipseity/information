Lab 7 Decision Trees and Random Forests

Introduction

This lab is inspired by the field of Machine Learning and, more precisely, the classification problem. In a nutshell, a classification problem consists of two phases. In the first phase (model training), given a Dataset of points and their corresponding labels, we train a model so that it can accurately predict their label. In the second phase (prediction), we provide unlabeled points (usually called "test points") to the trained model, and predict their label. Although multiple types of models exist (e.g., Neural Networks, linear classifiers), we will focus on decision trees and random forests. In this lab, we will give you already trained Decision Trees and Random Forests, and your task is to implement the second phase (prediction).

To simplify things, you can assume that the Decision Trees and Forests we provide have been trained using the Iris Dataset. In this setting, every test point is described by four integer values corresponding to Sepal Length, Sepal Width, Petal Length, and Petal Width. Furthermore, the given models classify a test point to three different species of the Iris family (Versicolor, Setosa, and Virginica).

More Details on Decision Trees and Random Forests

Decision Tree

Decision trees are one of the most popular data structures used in machine learning, data mining, and decision analysis. For classification tasks, a decision tree is nothing more than a tree-like data structure (as you learned in class). In this lab, we will limit our scope to the simplest form of a decision tree, which is a binary tree. Such a tree consists of two types of nodes (see Figure 1). The first type is the decision nodes, which contain a split attribute (e.g., Sepal Length, Sepal Width, Petal Length, or Petal Width) along with a split value. The second type of nodes are the end nodes (leaf nodes of the tree), which contain a label (in our case, flower species).

Making Predictions Using a Decision Tree: Making a prediction using a decision tree is similar to traversing a binary search tree. Let x be a data point where the i-th element corresponds to the i-th attribute. First, we start from the root denoted with v. If v is an end-node, then we return its label value. Otherwise, when v is a decision node, we compare the split attribute value of x with the split value. If x[v.split_attribute] <= v.split_value we move to the left child of v, and if x[v.split_attribute] > v.split_value we move to the right child. We continue recursively the same process with the new node. Figure 2 displays the prediction path we need to follow (presented with the red arrows and leaf), for x = (24,100,54,32).

Random Forest

Short Description: Although decision trees are heavily used in real-world applications, they tend to have poor prediction accuracies (this is caused by a problem known as overfitting). To overcome that issue researchers and practitioners use multiple trees (trained on slightly different datasets). The simplest model that stores multiple decision trees is Random Forests. Despite their simplicity in many situations, random forests have been proven to outperform more sophisticated models like neural networks in prediction accuracy. Figure 3 displays a random forest consisting of three decision trees.

Making Predictions Using a Random Forest: To make a prediction using a random forest, we use the Decision Tree prediction algorithm for every tree of the forest and finally compute the majority label (see the comments on the label description to see how to handle ties). Description of Classes

You are given three classes, the class Node representing a node of the tree, the class DecisionTree representing a decision tree and class RandomForest representing the random forest. More precisely: 

// Lies in DecisionTree.h file
class DecisionTree {
private:
    class Node{
    public:
        // split attribute of a (decision) node.
        //You can assume that the attribute is a positive number (>= 0).
        // If X is a data point of size N, then the i-th attribute will correspond to the i-th entry of X.
        int split_attribute;
        // split value of a (decision) node.
        int split_value;
        // Label of a (end) node
        int label;
        // Pointers to the children of the node
        DecisionTree *left = nullptr,*right = nullptr;
        // Constructors
        Node() = default;
        // Used when we want to initialize a decision-node
        Node(int split_value, int split_attribute) : split_attribute(split_attribute),split_value(split_value),label(0) {}
        // Used when we want to initialize an end-node
        Node(int label):split_attribute(0),split_value(0), label(label){}
        ~Node(){
            delete left;
            delete right;
        }

        // Task 1 (Step 1): Implement Tree prediction
        bool IsEndNode(){
        }
    };
   Node* root = nullptr;

public:

    // Constrcutors For a decision tree
    DecisionTree() = default;
    DecisionTree(Node *node)
    {
        root = node;
    }
    ~DecisionTree() {
        if (root != nullptr) {
            delete root;
        }
    }
    bool is_empty() const { return root == nullptr; }

    // Functions used to add decision and end nodes.
    // WARNING : Do not modify these methods
    // height : corresponds to the height (heigth = 0 is the root)
    // position : corresponds to the relative position of the node (a value between 0 and 2^height)
    // add a decision node with the given split value and split attribute at the position and height
    void AddDecisionNode( int attribute, int split_value, int &position, int &height);
    // add an end node with the given a label at the position and height
    void AddEndNode( int label, int &position, int &height);

    // Task 1 (Step 2): Implement Tree prediction
    int TreePredict(int *x);
};

// Lies in RandomForest.h file

class RandomForest {
  public:
    // Number of trees a forest has
    int NumberOfTrees;
    // An array of decision trees
    DecisionTree *Trees = nullptr;
    RandomForest() = default;
    RandomForest(int NumberOfTrees): NumberOfTrees(NumberOfTrees){
        Trees = new DecisionTree[NumberOfTrees];
    }
    ~RandomForest(){
      if(Trees != nullptr){
          delete[] Trees;
      }
    }
    void InitializeForest(int NumberOfTrees){
        this->NumberOfTrees = NumberOfTrees;
        Trees = new DecisionTree[NumberOfTrees];
    }
    // TODO 2: Implement Forest Prediction
    int ForestPredict(int *x);
};

Lab Tasks

Task 1: Decision Tree Prediction
  • Step 1:
    Implement the IsEndNode() method which returns true if a node is an End Node and false otherwise. Recall that an End node is a leaf of the tree.
  • Step 2:
    Implement the TreePredict(int *x) method in DecisionTree.cpp which takes as input an integer array x (of size 4), and outputs the predicted label. You can assume that every path of the tree eventually leads to an end-node and that the root of the tree will always be a Decision Node. However, you can make no assumptions about the depth of the tree and its structure (other than it is binary). (Hint: You can use recursion).
Task 2: Random Forest Prediction
    Implement the ForestPredict(int *x) method in RandomForest.cpp. You are advised to work in the following steps:
  • Step 1:
    Using TreePredict of DecisionTree, collect the predicted labels of every tree in the forest. You can assume that you will always have only three labels and that the label value of Node class will always be between 0 and 2.
  • Step 2:
    Use the collected predictions and find the most frequent one. In case of a tie, select the label with the smaller id. For instance, if two trees predicted 0, one tree predicted 1 and two trees predicted 2, then select the label with id = 0.
